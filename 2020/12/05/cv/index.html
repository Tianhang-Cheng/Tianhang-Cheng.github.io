Research Experience

1. Finding a new type of group equivariant CNNs

  • Introduction
    Traditional CNNs are equipped with translation equivariant, i.e. translating the input image to the CNNs, we will get a translated feature map. This feature of CNNs makes it robust in extracting the features of translational objects. However, CNNs are generally not equivariant to other transformations, such as scaling, reflection or rotation. Therefore, the performance of traditional CNN is very poor in recognizing rotating/scaling/reflection objects. To solve this problem, the Group Equivariant CNNs (G-CNNs) are introduced. With the help of group theory, G-CNNs can construct special convolution kernels that guarantee the equivariance of convolution，which has a high practical value.

  • What I am Doing Now
    I am running the code of the existing models on github to understand their implementation details. What’s more, I’m trying to design a new kind of G-CNN and came up with some ideas, but some have been discovered earlier by other authors. For example, I noticed that a NIPS2019 paper has proposed a general theory about E(2)(A group containing rotation & translation & reflection) Equivariant CNNS, it seems that G-CNN are already fully explored. However, there are many variations of G-CNNs that cannot be summarized by this theory, such as Oriented response networks，3D steerable CNNs，I am looking for inspiration from their paper. What’s more, I’m also trying to combine the attention mechanism with the G-CNNs to improve network performance.

2. Segment the 3D point cloud to achieve obstacle avoidance

  • Introduction
    The depth camera can be used to obtain 3D point cloud data, thereby obtaining rich depth information and object contour information. In this project, I used Intel’s RealSense depth camera to collect point cloud data, and then passed the data to the mobile app for image segmentation and obstacle prediction, thereby providing blind people with a mobile phone guide function.

  • What I Have Done
    I wrote an APP on Android Studio to read the data passed by realsense, and then run the open source segmentation algorithm on the phone to judge whether there are obstacles in the field of view. If there is an obstacle, the user will be prompted by voice.
